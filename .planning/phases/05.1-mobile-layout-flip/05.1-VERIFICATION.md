#### 2. Gallery Overlay Opens Correctly

**Test:** Tap the Gallery button on mobile.
**Expected:** Full-screen overlay opens showing the canvas grid in 2 columns. The overlay has an X close button in the canvas toolbar (top-right area, mobile only).
**Why human:** Overlay open/close state change and visual layout correctness are runtime behaviors.

#### 3. Gallery Close + Focus Return

**Test:** While the gallery overlay is open, tap the X close button.
**Expected:** Overlay closes, generator view (sidebar contents) returns. Pressing Tab once should move focus to the Gallery button in the mobile top bar.
**Why human:** Focus management via `queueMicrotask` requires a live browser to verify.

#### 4. Desktop Zero Regression

**Test:** Set viewport to >= 1024px. Observe the full layout.
**Expected:** No mobile top bar visible. No Gallery button. No X close button on canvas toolbar. Sidebar (32rem) on left, canvas full-width on right — identical to pre-Phase-5.1 desktop.
**Why human:** Visual breakpoint boundary check requires inspection at exactly 1023px → 1024px.

#### 5. Language Toggle on Gallery Button

**Test:** On mobile with gallery closed, use the language switcher in the sidebar account row.
**Expected:** Gallery button label flips between "Galeri" (Turkish) and "Gallery" (English) in real time.
**Why human:** i18n re-render requires runtime verification.
