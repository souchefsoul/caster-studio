# Phase 5.1: Mobile Layout Flip ‚Äî Generator Primary + Gallery Button - Context

**Gathered:** 2026-04-19
**Status:** Ready for planning
**Source:** Pivot captured during Phase 5 real-device validation ‚Äî user chose to restructure mobile layout so the generator is primary and the canvas lives behind a Gallery button.

<domain>
## Phase Boundary

On phones, flip the primary mobile view from the canvas to the generator (everything currently in the sidebar: navigation, mode panels, prompt, controls, brand face, account). The canvas (grid + single view built in Phase 5) becomes a full-screen overlay opened via a "Gallery" button in the top bar. Desktop (`lg+`) is unchanged. Phase 5 artifacts (grid overlay, single-view fill, iOS video attributes) stay and are reused inside the gallery overlay.

</domain>

<decisions>
## Implementation Decisions

### Primary Mobile View
- Below `lg`: the generator UI (current sidebar contents) renders inline as the primary full-width view ‚Äî NOT behind a drawer/hamburger anymore
- Phase 4's drawer machinery (backdrop, `translate-x-full`, focus-return on close) is retired for the sidebar role on mobile ‚Äî sidebar is always visible on mobile by default
- The sidebar's existing internals (`BrandFacePanel`, `OnModelPanel`, `CatalogPanel`, `ColorwayPanel`, `DesignCopyPanel`, `VideoPanel`, `PromptPanel`, `GenerationControls`, account row) are unchanged; only the container's layout role changes

### Gallery Button
- Lives in the top bar area of the mobile layout, replacing/superseding the hamburger role on mobile
- Icon + label: use a gallery/images icon from `lucide-react` (e.g., `Images` or `LayoutGrid`); label "Gallery" is Claude's discretion
- Visible only below `lg`; hidden on `lg+` (no desktop change)
- Tap: opens the canvas (grid + single view) as a full-screen overlay
- Close affordance: an explicit back/close button inside the gallery overlay returns to the generator view and returns focus to the Gallery trigger

### Desktop (lg+)
- Unchanged ‚Äî current sidebar + canvas side-by-side layout is preserved
- All Phase 5.1 changes must gate on `lg:` modifiers so desktop sees zero regression

### Phase 5 Work
- Retained as-is ‚Äî grid 2/3/4 columns, always-visible mobile overlay, single-view `p-0 lg:p-4`, `playsInline muted` on video ‚Äî all of this continues to work inside the gallery overlay

### Claude's Discretion
- Exact mechanism for "sidebar is primary on mobile" ‚Äî options:
  - (A) Drop the `translate-x-full` drawer pattern below `lg` and always render sidebar in document flow
  - (B) Keep sidebar fixed, set `sidebarOpen` default to `true` on mobile AND remove backdrop on mobile
  - (A) is cleaner; planner picks the path that minimizes diff risk to Phase 4
- Whether the "Gallery" trigger lives inside the sidebar header, inside a new top bar, or replaces the current account row ‚Äî planner decides based on minimal-change path; CONTEXT says "top bar"
- Whether state is a new `galleryOpen` slice in `useWorkspaceStore` or the existing `sidebarOpen` is inverted-in-role on mobile ‚Äî planner picks; a clean new `galleryOpen` is preferable for readability
- Transition: instant toggle is acceptable; if an animation is added it should match the Phase 4 "150ms-ish" convention ‚Äî no flashy animation

</decisions>

<specifics>
## Specific Ideas

- User watched the Phase 5 live demo on their phone and asked to flip the mobile model ‚Äî the canvas-first pattern felt wrong; the generator-first pattern matches a phone-native mental model ("I came here to make something, show me the input first; the gallery is secondary")
- Keep Phase 5 grid + single view + iOS video work ‚Äî it's reused inside the gallery overlay; do NOT revert any of it

</specifics>

<deferred>
## Deferred Ideas

- Swipe-right-to-open-gallery gesture ‚Äî deferred; a tap on the Gallery button is enough for v1.1
- Persistent gallery-open state across sessions ‚Äî not needed; default to closed on load
- Gallery preview count badge on the Gallery button (e.g., "12") ‚Äî nice-to-have, skip for v1.1
- Animation for gallery overlay enter/exit beyond a simple ~150ms transition ‚Äî keep it snappy, not flashy

</deferred>
