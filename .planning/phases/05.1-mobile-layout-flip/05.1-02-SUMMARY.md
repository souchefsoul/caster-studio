## Decisions Made

- **Gallery button in a new mobile-only top bar:** Keeps the trigger always visible as primary navigation from the generator view; avoids cluttering the sidebar and avoids confusion over whether gallery is a "mode" or a "view."
- **Close button reuses the retired Canvas hamburger slot:** The Canvas toolbar already has safe-area inset + flex-wrap + `lg:hidden` from Phase 5; no new overlay header needed. `data-sidebar-trigger` was retired in Plan 01 so repurposing the slot with `data-gallery-close` is a net simplification.
- **`data-*-trigger` / `data-*-close` attribute pair over React refs:** WorkspaceLayout and Canvas stay decoupled — no `forwardRef`, no context provider, no ref threading. Mirrors the Phase 04-02 focus-return contract.
- **`queueMicrotask` over `setTimeout(0)`:** Synchronous-with-React-commit; runs after overlay unmount and before browser paint, eliminating the 16ms flash `setTimeout(0)` can introduce.
- **Retired drawer backdrop + auto-close `useEffect` in the same edit:** Both became dead code the moment the sidebar became primary-in-flow in Plan 01. Keeping them would have bloated this diff with no behavioral benefit and would have surfaced as "why is this still here?" review noise.
- **Desktop byte-preserved via two sharp gates** (`lg:flex-row` on root, `hidden lg:flex` on canvas column): No breakpoint-conditional content inside a shared container — every mobile-only element is `lg:hidden`, every desktop-only element is `hidden lg:flex`. Two directions, never mixed.
