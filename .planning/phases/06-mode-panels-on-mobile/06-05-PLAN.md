
   Note: `generations` is ordered newest-first by the store (existing behavior — used elsewhere in Canvas). `.find` gets the first (= newest) completed video.

   Then render the inline player block at the bottom of the JSX, BEFORE the root `</div>` (i.e. after the Generate button's closing tag):

   ```tsx
   {/* Inline player — most recent completed video */}
   {latestVideoGeneration?.imageUrl && (
     <div className="flex flex-col gap-1.5">
       <p className="text-xs font-medium">{t('workspace.canvas.videoReady')}</p>
       <video
         key={latestVideoGeneration.id}
         src={latestVideoGeneration.imageUrl}
         controls
         playsInline
         muted
         className="w-full rounded-none border border-border bg-black"
         style={{ aspectRatio: (latestVideoGeneration.params?.aspectRatio ?? '16:9').replace(':', ' / ') }}
       />
     </div>
   )}
   ```

   Rationale for each attribute:
   - `key={latestVideoGeneration.id}` — forces React to remount the <video> when a new generation completes so the new src is picked up and played from the start instead of staying paused on the old frame.
   - `controls` — MODES-05 requires "generated video plays in-panel" with user control.
   - `playsInline` — iOS Safari: prevents the video from being forced into fullscreen on play (CANVAS-05 locked contract).
   - `muted` — iOS Safari: required to allow any autoplay-like behavior; defensive default so the element never trips the iOS audio policy when controls replay the video. User can unmute via the controls.
   - `className="w-full rounded-none border border-border bg-black"` — panel-width, flat-border aesthetic (Win95), black background behind aspect-letterbox if the natural video doesn't fill the ratio.
   - `style={{ aspectRatio: ... }}` — applies CSS `aspect-ratio` based on the stored generation's aspectRatio. '16:9' → '16 / 9' (CSS aspect-ratio syntax). Fallback '16:9' if params.aspectRatio is missing. This constrains the player's rendered height to match the generated video's ratio, so layout doesn't jump when video loads its intrinsic size.

Do NOT add autoPlay — the user explicitly generated the video; tapping play on the controls is the intentional interaction, and autoPlay+muted on a mobile sidebar would be noisy UX. The CANVAS-05 contract Phase 5 set was "inline playback WITH controls, no forced fullscreen" — autoplay was on Canvas's single-view (a dedicated playback surface); here in the sidebar panel, user-initiated play via controls is the correct default.
