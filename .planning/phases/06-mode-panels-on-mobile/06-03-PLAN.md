Edit `src/components/ColorwayPanel.tsx`. Apply ALL of the following, and ONLY the following:

1. **Switch the chip container from a vertical column to a wrap flex** (around line 55):
   - From: `<div className="flex flex-col gap-1">`
   - To:   `<div className="flex flex-wrap gap-1">`

   This keeps existing desktop usability (max 4 chips — at lg+ they still fit on one or two rows) and eliminates vertical-list whitespace on mobile.

2. **Tighten the chip itself so it acts as a compact inline chip** (around lines 56–70):
   Current chip JSX:
   ```tsx
   <div
     key={index}
     className="flex items-center justify-between border border-border bg-muted/50 px-2 py-1"
   >
     <span className="text-xs">{color}</span>
     <button
       type="button"
       onClick={() => removeColor(index)}
       className="text-muted-foreground hover:text-foreground"
       disabled={colorwayColors.length <= 1}
     >
       <X className="size-3" />
     </button>
   </div>
   ```

   Replace with:
   ```tsx
   <div
     key={index}
     className="flex items-center gap-1 border border-border bg-muted/50 px-2 py-1"
   >
     <span className="text-xs">{color}</span>
     <button
       type="button"
       onClick={() => removeColor(index)}
       className="flex h-10 w-10 items-center justify-center text-muted-foreground hover:text-foreground disabled:opacity-50 disabled:cursor-not-allowed"
       disabled={colorwayColors.length <= 1}
       aria-label="Remove color"
     >
       <X className="size-3" />
     </button>
   </div>
   ```

   The 40x40 (h-10 w-10) button gives a tap-sized remove target without growing the chip's visible text footprint — the X icon remains size-3 (12px) but the hit area is 40px.

3. **Make the add-color input 16px font-size + 40px min height** (around lines 77–83):
   Current input:
   ```tsx
   <input
     type="text"
     value={newColor}
     onChange={(e) => setNewColor(e.target.value)}
     onKeyDown={handleKeyDown}
     placeholder={t('workspace.colorway.colorPlaceholder')}
     className="flex-1 border border-border bg-background px-2 py-1 text-xs rounded-none"
   />
   ```

   Change the className to:
   ```tsx
   className="flex-1 rounded-none border border-border bg-background px-2 py-1 text-base min-h-10"
   ```

   Rationale: `text-base` is Tailwind's 16px — iOS Safari's zoom-on-focus heuristic triggers below 16px, so this prevents the mobile viewport from zooming when the user taps the color input. `min-h-10` gives a 40px tap-target floor. Desktop renders the input at 16px base size — the previous text-xs (12px) was too small for touch anyway.

4. **Enlarge the Add button to 40px tap target** (around lines 84–89):
   Current:
   ```tsx
   <Button
     variant="outline"

     onClick={addColor}
     className="rounded-none text-xs px-2 py-1 h-auto"
   >
     {t('workspace.colorway.addColor')}
   </Button>
   ```

   Change className to:
   ```tsx
   className="rounded-none text-xs px-3 min-h-10"
   ```

   (Drop `py-1 h-auto` since `min-h-10` enforces the target and padding comes from Button defaults.)

Do NOT change: MAX_COLORS constant, addColor/removeColor/handleKeyDown logic, the ImageUpload block, the workspace.colorway.hint final paragraph, the empty-state / max-colors conditionals, or any store subscriptions.
