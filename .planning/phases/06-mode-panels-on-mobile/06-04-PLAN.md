<objective>
Make the Design Copy panel mobile-ready per MODES-04: reference image + modifications textarea stay stacked vertically (CONTEXT decision), the textarea auto-grows with content via CSS `field-sizing: content` (Tailwind v4 `field-sizing-content` utility), minimum 3 rows, font-size 16px+ to prevent iOS auto-zoom.

Purpose: MODES-04 requires users to edit the modifications textarea without "input zoom or layout break." The current textarea uses `font-mono text-sm` (14px) — iOS Safari will auto-zoom on focus. It also uses `resize-none` with no growth mechanism, so long prompts either scroll inside a fixed 3-row box or — if the developer ever removes resize-none — overflow horizontally.

`field-sizing: content` is the modern CSS solution that lets the textarea height track its content exactly, no JS handlers needed. Tailwind v4 ships `field-sizing-content` and `field-sizing-fixed` utilities. No polyfill — Vite + React 18 on modern iOS Safari + Android Chrome supports it (Safari 17.4+, Chrome 123+). Phase 8 real-device verification will catch any remaining gap.

Output: A Design Copy textarea that grows as the user types, keeps a 3-row floor, and never triggers iOS zoom.
</objective>
